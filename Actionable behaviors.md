This quick recall table summarizes all key concepts regarding **Actionable Behaviors** within a BPO and QA context, including their definition, importance, specific categories, and practical application.

### **Actionable Behaviors Recall Table**

| Category                           | Key Concepts, Characteristics, and Examples                                                                                                                                                                                                                                                                               |
| :--------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Core Definition**                | Specific, observable, and repeatable actions or habits that directly impact performance outcomes. They are the "how" behind a performance gap.                                                                                                                                                                            |
| **Key Characteristics**            | Actionable behaviors must be **controllable, visible, measurable, and coachable**.                                                                                                                                                                                                                                        |
| **Why They Matter**                | • **Translate data into coaching:** Moves from vague metrics (e.g., "low CSAT") to specific actions.<br>• **Create accountability:** Focuses on conduct rather than personality.<br>• **Fair QA scoring:** Links outcomes to observable conduct.<br>• **Consistency:** Defines exactly what "good" looks like for agents. |
| **1. Knowledge & Training**        | • **Behavior:** Following step-by-step SOPs, using correct product info, referencing KB tools.<br>• **Coaching:** Use checklists, knowledge refreshers, or modeling tool usage.                                                                                                                                           |
| **2. Communication & Soft Skills** | • **Behavior:** Warm greetings, active listening (no interruptions), summarizing resolutions.<br>• **Coaching:** Role-play openings, call playback analysis, and adding confirmation steps to scripts.                                                                                                                    |
| **3. Process & Compliance**        | • **Behavior:** ID verification, accurate CRM documentation, following escalation protocols.<br>• **Coaching:** Reviewing scripts, auditing ticket notes in huddles, and using visual flowcharts.                                                                                                                         |
| **4. Systems & Tools**             | • **Behavior:** Efficient tool navigation (minimal toggling), correct data entry, prompt IT issue reporting.<br>• **Coaching:** Sharing keyboard shortcuts, data-entry simulations, and reinforcing escalation paths.                                                                                                     |
| **5. Behavioral & Motivation**     | • **Behavior:** Punctuality/adherence, accepting feedback constructively, taking ownership of errors.<br>• **Coaching:** Reinforcing team reliability, using "feedforward" models, and recognizing honesty.                                                                                                               |
| **6. Environment & Operations**    | • **Behavior:** Logging in early for prep, reporting high volume/slowness, maintaining a quiet workspace.<br>• **Coaching:** Encouraging readiness routines and sharing WFH best practices.                                                                                                                               |
| **The Analyst’s Framework**        | To identify a behavior, ask:<br>1. What specific behavior led to this result?<br>2. Is it observable, measurable, and coachable?<br>3. Can we track improvement through scoring or feedback?                                                                                                                              |
| **Practical Application**          | **Vague:** "Improve empathy."<br>**Actionable:** "Acknowledge the customer's concern before providing a solution (e.g., 'I can understand how that would be frustrating.')".                                                                                                                                              |
